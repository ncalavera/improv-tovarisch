# Инструкция для исследования импровизационных форматов

## Задача
Исследуй импровизационный формат и верни данные в структурированном JSON формате на русском языке.

## Что нужно найти

### 1. Базовая информация (обязательно)
- **name**: Название формата (на русском, с оригинальным названием если нужно)
- **shortDescription**: Краткое описание в одно предложение (20-40 слов)
- **fullDescription**: Полное описание формата (3-5 абзацев), разделенные `\n\n`:
  - История создания и авторы
  - Основная структура
  - Ключевые особенности
  - Философия формата

### 2. Параметры (обязательно)
- **minPlayers**: Минимальное количество игроков (число)
- **maxPlayers**: Максимальное количество игроков (число)
- **duration**: Длительность в минутах (строка, например "20-25 минут")
- **difficulty**: Уровень сложности (выбрать одно):
  - `"beginner"` — начальный
  - `"intermediate"` — средний
  - `"advanced"` — продвинутый

### 3. Структура (обязательно)
**components**: Массив компонентов формата. Каждый компонент:
```json
{
  "name": "Название этапа",
  "description": "Что происходит",
  "duration": "2-3 минуты"
}
```

### 4. Правила и навыки (обязательно)
- **montageRules**: Правила монтажа/перехода между сценами (1-2 предложения)
- **skills**: Массив развиваемых навыков (3-8 слов через дефис)
  - Примеры: `"слушание"`, `"да-и"`, `"связи"`, `"паттерны"`, `"быстрота-мышления"`
- **focus**: Педагогический фокус формата (1-2 предложения)

### 5. Связи (если есть)
- **variations**: ID форматов-вариаций (массив строк, например `["1", "6"]`)
- **prerequisites**: ID форматов-предпосылок (массив строк)
- **similarTo**: ID похожих форматов (массив строк)

### 6. Ресурсы (если есть)
- **sourceVideos**: Массив URL видео примеров
- **notes**: Дополнительные заметки, советы, частые ошибки, история в России (3-5 предложений)

## Расширенная информация (для сложных форматов)

### Если формат требует детального описания, создай отдельный `-extended.json` файл:

#### **openings** (для форматов с открытиями)
Массив объектов открытий:
```json
{
  "name": "Название опенинга",
  "description": "Как проходит (2-3 предложения)",
  "howItWorks": "Детали механики (2-3 предложения)",
  "threeRounds": "Описание трех раундов (если применимо)",
  "structure": "Структура с использованием **bold** для ключевых терминов",
  "example": "Конкретный пример",
  "result": "Что происходит после"
}
```

#### **keyRules** (ключевые правила)
Массив строк с правилами. Используй:
- `**bold**` для важных терминов
- `*italic*` для техник

Пример: `"Не строй сюжет — ищи **рифмы и паттерны**"`

#### **example** (пример выступления)
```json
{
  "theme": "Тема выступления",
  "opening": {
    "type": "Тип опенинга",
    "description": "Что произошло в опенинге с **bold** терминами"
  },
  "scenes": [
    {
      "beat": "A1",
      "description": "Что произошло в сцене",
      "game": "Какая игра/паттерн",
      "theme": "Тема (опционально)",
      "resolution": "Разрешение (опционально)",
      "finalLine": "Финальная реплика (опционально)"
    }
  ]
}
```

#### **resources** (ресурсы)
```json
{
  "videos": [
    {
      "title": "Название видео",
      "url": "https://...",
      "lang": "RU" или "EN"
    }
  ],
  "books": [
    {
      "title": "Название книги",
      "authors": "Авторы",
      "description": "Краткое описание",
      "url": "https://... (опционально)"
    }
  ],
  "links": [
    {
      "title": "Название ресурса",
      "url": "https://..."
    }
  ]
}
```

Создан в конце 1980-х в iO Theater.\n\nКлючевая особенность — импровизаторы не пересказывают историю буквально, а вдохновляются эмоциями, темами и образами из монологов.",

```json
  "skills": ["слушание", "вдохновение", "монтаж", "эмоциональная-работа"],
  "focus": "Слушание и интерпретация реальных историй в сценах",
  "variations": [],

```

```json
{

```

## Важные правила

1. **Весь текст на русском языке** (кроме оригинальных названий в скобках)
2. **Используй `\n\n`** для разделения абзацев в fullDescription
3. **Используй markdown** в extended файлах:
   - `**bold**` для важных терминов
   - `*italic*` для техник и методов
4. **Проверяй факты** из нескольких источников
5. **Указывай URL** только реально существующих ресурсов
6. **ID форматов** — числа в виде строк: `"1"`, `"2"`, `"3"`
